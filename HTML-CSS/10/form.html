  • < form >태그는 데이터를 전송하기위해 묶어주는 역활을 하는 태그로 submit을 이용하여 전송한다.
  • html 페이지 뿐만 아니라 여러가지 서버프로그램을 호출 및 데이터를 전송할 수 있다.
  • 속성
    • name : 폼의 이름
    • action : 대상 url
    • method : 전송 방법
      • GET
        • 전송데이터가 보임 : URL에 ?name=value & name=value 형식으로 전송된다.
        • 데이터의 길이 제한이 있다.(URL길이 제한에 따르며 이는 브라우저, 서버에 따라 다르다)
        • 보안에 취약하다. (민감한 데이터(개인정보,암호 등)은 절대 사용하지 말것!)
        • POST에 비해 사용하기 편하고, 속도도 빠르다.
      • POST
        • 전송데이터가 안보임 : body안에 포함되어 전송된다.
        • 데이터 길이제한이 없다.
        • 보안이 우수하다.
        • GET에 비해 사용하기 어렵다.
